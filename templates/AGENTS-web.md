# Agent Instructions: Web Project

## Общие правила

- Сначала прочитай `README.md`, `.cursor/rules/` и существующие UI-паттерны.
- Сохраняй текущий стиль проекта, routing, state-management и UI kit.
- Не добавляй зависимости без понятной пользы.
- Не храни секреты во frontend-коде.

## Web UI

- Компоненты должны быть маленькими и понятными.
- Разделяй data fetching, бизнес-логику и отображение, если структура проекта это поддерживает.
- Обрабатывай loading, empty, error и success states.
- Формы должны иметь labels, понятные ошибки и базовую accessibility-поддержку.
- Клиентская валидация нужна для UX, но безопасность должна проверяться на backend.

## Проверка

- Запускай formatter, typecheck и тесты, если они есть.
- Если меняется пользовательское поведение, обновляй `README.md` или changelog по правилам проекта.
