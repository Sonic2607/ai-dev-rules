# Agent Instructions: Go Project

## Общие правила

- Сначала прочитай `README.md`, `.cursor/rules/` и соседний код.
- Делай минимальные безопасные изменения.
- Не переписывай архитектуру без согласованного плана.
- Не добавляй секреты в код, конфиги, тесты и логи.

## Go

- Entry point держи в `cmd/<service>/main.go`.
- Бизнес-логику держи в `internal/`.
- `main.go` должен только грузить конфиг, настраивать логирование, создавать приложение, запускать компоненты и ждать shutdown.
- Используй `context.Context` для долгоживущих операций.
- Обрабатывай `SIGINT` и `SIGTERM`.
- Ошибки возвращай с полезным контекстом.
- Запускай `gofmt`, `goimports` и `go test ./...`, если это возможно.

## Логи и конфигурация

- Конфигурация должна приходить из environment variables.
- Логи должны быть JSON в stdout.
- Не логируй токены, пароли, ключи, PII и полные чувствительные payload.
