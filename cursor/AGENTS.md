# Agent Instructions

Этот проект управляется с помощью AI-агентов. Работай аккуратно: сначала читай код, потом меняй.

## Общие правила

- Отвечай на языке пользователя.
- Делай минимальные безопасные изменения.
- Не переписывай несвязанный код.
- Не удаляй и не откатывай чужие изменения без явного запроса.
- Не добавляй секреты в репозиторий.
- Сохраняй существующий стиль проекта.

## Перед изменениями

1. Прочитай `README.md`.
2. Найди локальные правила проекта: `AGENTS.md`, `CLAUDE.md`, `.cursor/rules/`.
3. Проверь существующие паттерны в соседнем коде.
4. Коротко объясни подход и список файлов, которые будут изменены.

## Код

- Предпочитай простые решения.
- Обрабатывай ошибки явно.
- Не добавляй абстракции "на будущее".
- Держи бизнес-логику вне entrypoint-файлов.
- Добавляй тесты там, где изменение затрагивает важную логику.

## Инфраструктура

- Изменения должны быть идемпотентными.
- Используй переменные вместо machine-specific значений.
- Для сервисов предпочитай systemd, env-файлы, healthcheck и понятные логи.
- Перед restart/reload проверяй конфиги, если есть валидатор.

## Проверка

После изменений запусти доступные проверки проекта: форматирование, lint, тесты или dry-run. Если проверку нельзя выполнить, явно напиши почему.
